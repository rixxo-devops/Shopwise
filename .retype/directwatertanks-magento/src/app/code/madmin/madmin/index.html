# Madmin_Madmin

Magento 2 module that provides read-only API permissions for REST API endpoints.

# Overview

This module creates read-only versions of all ACL resources, allowing you to grant API users GET-only access to endpoints without giving them full permissions. This is useful for integrations that only need to read data from your Magento store.

# Features

  • Automatic ACL Duplication: Creates read-only versions of all ACL resources with _api_ro suffix
  • GET Request Authorization: Allows GET requests through with read-only permissions when full permissions would normally be required
  • Admin UI Integration: Adds "API (Read Only)" section in Admin > System > Permissions > User Roles
  • Security Checks: Maintains HTTPS requirements and backpressure limits

# How It Works

# ACL Resource Generation

The AclResourceProviderPlugin duplicates all existing ACL resources with:

  • ID suffix: _api_ro (e.g., Magento_Catalog::products_api_ro)
  • Title suffix: (RO) (e.g., "Products (RO)")
  • Organized under: Madmin_Madmin::api_readonly group

# Request Validation

The RequestValidatorPlugin intercepts authorization failures for GET requests:

  1. Checks if the request is a GET request
  2. Converts required ACL resources to read-only versions
  3. Authorizes the request if user has read-only permissions
  4. Enforces HTTPS and backpressure requirements

# Usage

  1. Enable the module:

    bin/magento module:enable Madmin_Madmin
bin/magento setup:upgrade

  2. Create API role in Admin:

    • Navigate to System > Permissions > User Roles
    • Create a new role
    • Under Role Resources, expand "API (Read Only)"
    • Select the read-only permissions needed

  3. Assign to API user:

    • Create or edit an integration user
    • Assign the read-only role
    • User can now make GET requests to those endpoints

# Example

Without this module:

  • User needs Magento_Catalog::products to GET /rest/V1/products

With this module:

  • User only needs Magento_Catalog::products_api_ro to GET /rest/V1/products
  • POST/PUT/DELETE requests still require full permissions
