# Kingfisher Direct Magento

This repository contains Magento, themes and modules backing the customised Magento setup

# Prerequisites

For a quicker start there is Docker setup.

  • Docker
  • docker-compose
  • awscli (configured) - I have v1, but it should work with v2
  • node - >= LTS version

# Preparing Environment

  1. Clone this repository
  2. Clone docker compose files to folder outside this repository
    git clone https://gitlab.com/lumnn/magento2-docker-compose.git
  3. Navigate to repository from step 2 and generate self-signed ssl certificate by running
    bin/create-self-signed-ssl
  4. Ensure composer config directory is existing on your system
    mkdir -p ~/.config/composer
  5. Navigate to folder where repository from step 1 was cloned into.
  6. Create .env file for docker compose cp .env.sample .env
  7. Update .env file with correct path to docker-compose.yml from repo cloned in step 2.
  8. Start environment
    docker compose up
  9. Create magento env.php file
    bin/setup-magento

This is not everything. You'll need to also follow steps below.

# Starting / Updating environment

Above preparing environment steps are one-time only steps. Following ones you will execute more often to keep your environment updated.

  1. Install composer dependencies docker compose exec php composer install

  2. Import database

DB_DUMP_FILENAME=db_$(date +%s).sql \
    && aws s3 cp s3://backup.kingfisherdirect/magento/live.sql $DB_DUMP_FILENAME \
    && bin/import-db $DB_DUMP_FILENAME

  1. Run upgrade after importing database or installing newer composer dependencies
    docker compose exec php bin/magento setup:upgrade

  2. Compile themes stylesheets

    • pushd src/src/directwatertanks-theme && npm install && popd
    • pushd src/src/kingfisherdirect-theme-2020 && npm install && popd
    • pushd src/src/gritbins-22 && npm install && popd

In order to work with themes jump to Theme Development section

# Working in environment

After the Installation steps are completed you can start developing. To stop environment simply CTRL + C on the docker-compose up command.

Next time to start project simply run docker-compose up and you're ready to go. Time to time you may need to install latest composer changes or update your database with more recent live website data.

# Emails

After starting docker-compose there is Mailcatcher service running to find it's url run:

  1. docker compose ps
  2. Find line with mailcatcher
  3. Find IP:Port that points to 1080 (for example 0.0.0.0:32775->1080), which states port to use (in this example 32775). Then navigate to either http://0.0.0.0:PORT or http://localhost:PORT

# Images

By default magento is setup to load live images. Although it's alright for most cases sometimes it can cause issues. To get images working locally you need to follow these steps:

  1. Paste theese lines in env/magento.local.env. You may add more if there is more websites
CONFIG__DEFAULT__WEB__SECURE__BASE_MEDIA_URL=media/
CONFIG__WEBSITES__DIRECTWATERTANKS__WEB__SECURE__BASE_MEDIA_URL=media/
CONFIG__WEBSITES__KINGFISHERDIRECT__WEB__SECURE__BASE_MEDIA_URL=media/
  1. Restart containers
  2. Download images using bin/download-media. Script should guide you through the process

# IDE Set-up

# XML Lint and Completitions

Magento is using XML file format for configurations. Setting up a XML linter or getting auto completitions is not straight forward for everything else than PHPStorm.

In order to use that with SublimeText or VSCode you can follow these steps.

VS Code

  1. Install this extension - https://marketplace.visualstudio.com/items?itemName=redhat.vscode-xml
  2. Run bin/generate-urn with docker-compose beeing up
  3. Open Extension Settings panel in VS Code
  4. Switch to Workspace tab (scope)
  5. Add to "XML: Catalogs" a path src/var/magento_urn_oasis.xml

Sublime Text

  1. Instal packages: LSP and LSP-lemminx
  2. Ensure you opened project folder as Sublime Text project, not just by open folder. (You can create project, by opening folder and then: Project > Save Project As..)
  3. Run bin/generate-urn with docker-compose beeing up
  4. Configure you .sublime-project by merging this JSON into existing contents
{
    "settings":
    {
        "LSP":
        {
            "LemMinX": {
                "settings": {
                    "xml": {
                        "catalogs": [
                            "src/var/magento_urn_oasis.xml"
                        ]
                    }
                },
            }
        },
    },
}

# PHP CodeSniffer

Read about PHP Coding Standard on Magento Docs

Setting this up will enforce same as in Magento rules.

VS Code

  1. Install this extension https://marketplace.visualstudio.com/items?itemName=obliviousharmony.vscode-php-codesniffer
  2. Install PHP in similar as bin/cli php -v version on your system globally
  3. Open Extension Settings panel in VS Code
  4. Switch to Workspace tab (scope)
  5. Set Executable to: php src/vendor/bin/phpcs (php may be changed to php7 or similar on some systems)
  6. Set Standard to: Custom
  7. Set Standard Custom to: src/.phpcs.xml.dist

Sublime Text

  1. Install SublimeLinter and SublimeLinter-phpcs packages
  2. Merge this config into your *.sublime-project
{
    "settings":
    {
        "SublimeLinter.linters.phpcs.executable": ["docker-compose", "exec", "php", "vendor/bin/phpcs"],
        "SublimeLinter.linters.phpcs.working_dir": "${project_path}/src",
    },
}

# Phpactor

For PHP completitions and other useful helping features you can install phpactor by downloading VSCode extension and installing directly the .VSIX file https://github.com/phpactor/vscode-phpactor/releases/

Then you have to modify it's configuration to take most of it. Go to extension settings, switch to workspace tab and then set phpactor.config value using the following template. First path has to be absolute path to the src folder inside this project folder.

"file_path_resolver.project_root": "/home/<YOUR_NAME>/Projects/kfd-magento/src",
"language_server_php_cs_fixer.enabled": true,
"indexer.include_patterns": [
    "\/**\/*.php",
    "\/**\/*.phtml"
]

# Debugging

To start debugging you need to start docker with ENV variable XDEBUG_ENABLE=1.
To do that you can simply run XDEBUG_MODE=develop,debug docker-compose up.

To run debugger you need to trigger request with XDEBUG_TRIGGER variable sent with POST, GET or COOKIE. For this you may use Xdebug extensions for browsers.

Optionally you can configure it to start automatically XDEBUG_CONFIG="start_with_request=yes". This can be either passed to command or saved in your .env file.

Xdebug will work on localhost:9003 and that's how you should configure your IDE.

Example configuration for VS Code.

{
    "name": "XDebug",
    "type": "php",
    "request": "launch",
    "port": 9003,
    "pathMappings": {
        "/var/www/html": "/home/<YOUR_NAME>/Projects/kfd-magento/src"
    }
}

# Theme Development

At the moment themes lives in src/src/* folders. Ideally they should be split between repositories, however that would require time invested into setup and learning how to set it up in effective for everyone way.

Each theme at the moment comes with package.json - a NPM package definition file, which describes dependencies and the way how the theme should be built.

The theme build/dev process is based on 2 npm commands withing appropriate theme: - npm run build - to build theme entirely - npm run dev - to work on styles in watch mode

More theme-specific instructions should be described in themes README.md.

# Troubleshouting

It's easy to get something not working in magento, so here's a list of things to do in first place to resolve problems:

  1. Clean Cache - bin/magento cache:clear
  2. Check Logs - src/var/logs/debug.log
  3. Clean all generated files and caches - bin/advanced-clean
  4. Check if all docker containers are working - docker-compose ps
  5. Clean browser cache
  6. Run - bin/magento setup:upgrade
  7. Restart computer

# Improve speed

There may be few reasons for that. And also few solutions:

  • Ensure Magento cache is enabled (there is many of them)
  • Set to production mode if you're not developing anythingbin/magento deploy:mode:set production
  • Or at least to default modebin/magento deploy:mode:set default
  • Ensure caches are enabledbin/magento cache:status
  • Check Logs - src/var/logs/debug.log contains all logs
  • Upgrade to linux as Windows environment will handle both Magento and Docker MUCH slower. Not tested on macOS at all
  • Upgrade to SSD
